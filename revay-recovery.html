Why Agents Die

Understand the failure modes before you debug one
üö®
Primary killer: config.apply Replacing the entire config in one shot silently wipes channel tokens or gateway auth. The gateway restarts but is deaf ‚Äî it can't receive messages and there's no recovery path without manual intervention.

üíÄ Failure Mode 1 ‚Äî Config Self-Destruction

Agent uses config.apply (full replace) instead of config.patch. Missing fields wipe tokens. Gateway restarts broken. Agent is alive but unreachable.

üíÄ Failure Mode 2 ‚Äî Gateway Field Corruption

Changing gateway.port, gateway.auth.token, or gateway.mode breaks the localhost connection. Agent is running, receiving nothing.

üíÄ Failure Mode 3 ‚Äî Channel Token Wipeout

Setting channels.telegram.enabled = false or losing the bot token during a patch. Gateway alive, zero messages in.

üíÄ Failure Mode 4 ‚Äî Workspace Amnesia

MEMORY.md corrupted, overwritten, or git backup failing silently. Agent wakes with no identity ‚Äî same model, no continuity.

üíÄ Failure Mode 5 ‚Äî Stale Hash Patch Escalation

Patch fails due to stale baseHash. Agent "fixes" it by using config.apply with incomplete JSON ‚Äî kills everything it wasn't trying to touch.

Config Safety Rules

Non-negotiable. Burn these in.
üö´
NEVER use gateway config.apply Only exception: Evan explicitly says "replace the full config" and provides it verbatim. Even then, verify the JSON is complete before applying.
‚ö†Ô∏è
Always get the current hash before patching Run config.get, extract hash, pass it as baseHash in your patch. If you skip this, the patch will fail ‚Äî and if you then escalate to config.apply, you may wipe everything.

üîí Frozen Fields ‚Äî Never Touch Without Explicit Written Instruction

  • gateway.port Breaks localhost connection if changed
  • gateway.auth.token Auth token ‚Äî change = locked out
  • gateway.mode Changes fundamental operation mode
  • channels.telegram.botToken Bot dies if this is wrong or missing
  • channels.discord.token Discord bot dies if this is wrong or missing
  • channels.telegram.enabled Setting false = no Telegram messages
  • channels.discord.enabled Setting false = no Discord messages
  • channels.discord.allowFrom Removing Evan's ID = he can't reach you
  • channels.discord.guilds Removing guild = no group chat access
