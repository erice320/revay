Options
Build vs. buy vs. hybrid
Four approaches — each with different speed, cost, and control tradeoffs.
Recommended — Phase 2

🔨 Build: Playwright Pipeline

Full custom pipeline on our stack
Full control over output quality
Integrates with our release pipeline
Uses PRDs we already have
No vendor dependency or per-video cost
Requires engineering investment (8–12 weeks)
Playwright scripts need generation per feature
Quick Win — Phase 1

🎬 Buy: Guidde

AI-powered demo video platform
Fastest path to production quality
AI generates voiceover + callouts automatically
PM/QA records once, AI handles the rest
Still needs a human to do the initial recording
SaaS cost; less end-to-end automation

🗺️ Interactive Tours: Storylane / Arcade

Guided walkthrough platforms
Very polished output
Interactive (users can click through)
Not video — not embeddable in email/social
Requires human recording flow

🎞️ Programmatic: Remotion

React-based video composition
Extremely high production quality
Fully automatable and brandable
Higher engineering lift
Better for animated explainers than live UI demos
Phased Plan
Ship value fast,
automate over time
A three-phase approach that delivers immediate marketing relief while building toward full automation.
Phase 1
Weeks 1–6 · Quick Win

Guidde + Human Trigger

  • Deploy Guidde for the marketing team
  • PM or QA records a 2-min screen pass per new feature
  • AI generates voiceover script and callout annotations
  • Marketing reviews and publishes
  • Cuts manual effort by ~80% immediately
Phase 2
Weeks 6–18 · Automated Pipeline

Full Playwright Automation

  • Agent reads PRD → generates UI script + narration
  • Playwright runs on demo environment with humanized cursor
  • ElevenLabs voices the narration in sync
  • FFmpeg assembles with intro/outro + captions
  • MP4 delivered to marketing Notion/Slack for review
Phase 3
Ongoing · Full Automation

Release-Triggered Pipeline

  • Feature ships → webhook fires automatically
  • No human trigger needed — end-to-end pipeline
  • A/B test script tones (technical vs. benefit-led)
  • Auto-publish approved videos to YouTube, Wistia, docs
  • Marketing approves, not creates
Requirements
What we need from
each team to make this work
This pipeline is only as good as its inputs. Three dependencies need to be locked down.

📄 Product

  • Structured PRDs with consistent fields: feature name, user role, step-by-step user flow, value prop (1 sentence), URL path in app
  • A clear "feature shipped" signal — Notion status change, GitHub release tag, or equivalent
  • Agreement on target video length (60s? 90s?) to guide script generation
  • Phase 1: A PM or QA willing to do a 2-min recording pass per feature

📣 Marketing

  • Define where videos live: YouTube channel, Wistia library, product docs site
  • Provide branded intro/outro template (15 sec each)
  • Decide: human review before publish, or auto-publish for minor features?
  • Define voiceover tone and persona — technical, benefit-led, conversational?
  • Establish a Notion/Slack intake for video delivery

⚙️ Engineering / Infra

  • Stable demo environment always accessible — not wiped by dev deploys
  • Seeded with realistic, visually appealing test data (not empty screens)
  • Headless server with GPU or display capability for Playwright video recording
  • ElevenLabs API integration (already in place for Rev.io)
Discussion
Open questions
for this room
Before we scope this for Product, let's align on these five things.
1

Where do PRDs live today — Notion, Confluence, Google Docs? Are they structured consistently enough to parse automatically, or do they vary by team?

2

Do we have a stable demo environment with realistic data that isn't touched by dev deploys? If not, what would it take to create one?

3

What's the "feature shipped" signal today? When something goes live, how does anyone know? Is there a consistent process we can hook into?

4

What's the target format? 60-second social clips? 2-minute feature walkthrough? One format for all, or different lengths by feature complexity?

5

Who approves before publish? Does every video need a marketing sign-off, or can minor feature updates auto-publish once the pipeline is trusted?
